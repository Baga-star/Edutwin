# ⚡ Оптимизации производительности — v6

## Проблемы которые были исправлены

### 1. ✅ Стриминг (`stream: true`)
**Было:** Пользователь ждал 3–15 секунд пока модель сгенерирует весь ответ, только потом видел текст.  
**Стало:** Текст появляется сразу по мере генерации — как в ChatGPT. Реализовано через SSE (Server-Sent Events).

**Файл:** `src/services/aiService.ts` → `getAgentResponseStream()`  
**Файл:** `src/App.tsx` → стриминг встроен в `handleSendMessage`

---

### 2. ✅ Умный выбор модели по агенту
**Было:** Все агенты использовали `qwen3-235b-a22b` — самую тяжёлую модель (235B параметров).  
**Стало:**
- `Diagnostic`, `Tutor`, `Assessment`, `CognitiveModel` → `qwen3-30b` (в 5–8× быстрее)
- `Analytics`, `Prediction` → `qwen3-235b-a22b` (там нужна глубина)

**Переменная:** `VITE_QWEN_MODEL_FAST=qwen3-30b` в `.env`

---

### 3. ✅ In-memory кэш вместо Redis HTTP round-trip
**Было:** Каждый запрос к кэшу делал `fetch → backend → Redis → обратно` (+200–500ms).  
**Стало:** Кэш живёт в памяти (`Map`), Redis обновляется в фоне без блокировки.

**Файл:** `src/services/aiService.ts` → `memGet()` / `memSet()`

---

### 4. ✅ Логирование fire-and-forget
**Было:** `await logProgressToNocoDB(...)` — блокировало UI.  
**Стало:** Все записи в NocoDB, MinIO, Redis — фоновые, не блокируют чат.

---

### 5. ✅ Профиль в памяти
**Было:** `loadProfileFromRedis()` при каждом обновлении.  
**Стало:** `profileCache: Map<string, StudentProfile>` — мгновенный доступ.

---

## Итог по скорости

| Сценарий | До | После |
|---|---|---|
| Первый ответ агента | 3–15 сек | ~0.3 сек до первого токена |
| Ответы из кэша | 200–500ms | ~50ms (memory) |
| Конец диагностики | 15–20 сек | ~2–4 сек |
| Логирование NocoDB | блокирует | фоновое |

## Настройка

Добавь в `.env`:
```
VITE_QWEN_MODEL_FAST=qwen3-30b
```
Если `qwen3-30b` недоступен у твоего провайдера — попробуй `qwen3-14b` или `qwen3-7b`.
